DOC DB Documentation Workspace

Use this workspace to build documentation that is easy to read, easy to query, and safe for automated edits.

What You Can Do Here

  • Author docs as structured blocks instead of loose markdown
  • Search and retrieve content quickly through SQLite-backed indexing
  • Edit visually in the DOC webview while keeping deterministic source
  • Style key sections with scoped ids and classes

Authoring Standard

  1. Start each major page with one clear H1 and one concise purpose paragraph.
  2. Use H2 sections for workflow steps, decisions, and references.
  3. Prefer bulleted lists for scanability and code blocks for exact examples.
  4. Keep section names task-oriented so AI retrieval stays semantically clean.

First 10 Minutes

  1. Open examples/welcome.dx in the DOC editor and read it in both Read and Edit mode.
  2. Create one new section with a heading, one paragraph, and one list.
  3. Assign ids to important blocks that may need future styling.
  4. Run ingestion or query tools to confirm retrievability before publishing.
